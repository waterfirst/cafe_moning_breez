# 분석 대시보드 사용법

## 접속
서버 실행 후 사장님 노트북 브라우저에서:
```
http://[내IP]:8000/dashboard?token=[관리자토큰]
```
(IP·토큰은 run.py 실행 시 검은 창에 표시됨. 즐겨찾기 추천)

⚠️ 매출 데이터이므로 **토큰 없이는 접근 불가**. 손님은 못 봅니다.

## 보이는 것

### 상단 KPI (오늘)
- 오늘 주문 수 / 매출 / 객단가 / 피크 시간

### 차트 6종
1. **당일 시간대별 주문량** — 오늘 몇 시에 바쁜지 (재료·인력 배치)
2. **오늘 인기 메뉴 TOP5** — 잘 나가는 메뉴
3. **요일별 평균 매출** — 주말/평일 패턴 (주말은 주황색)
4. **날씨별 일평균 주문수** — 비 오는 날 손님 감소폭 확인
5. **월별 매출 추이** — 시즌성 파악

## 활용 예시
- "16시가 피크네 → 15시 30분에 재료 미리 준비"
- "비 오는 날 30% 줄어드네 → 우천 시 발주 줄이기"
- "주말이 1.5배네 → 주말 알바 1명 추가"
- "여름에 아이스티 급증 → 6월부터 재고 늘리기"

## 자동 갱신
1분마다 자동 새로고침. 영업 중 띄워두면 실시간 현황판이 됩니다.

## 다음 단계 (4단계 재료 예측)
이 분석 데이터 위에서 "내일 날씨·요일 기준 예상 판매량 → 재료 준비량"을
자동 추천하는 기능을 붙일 예정입니다. 1년 데이터가 쌓일수록 정확해집니다.
