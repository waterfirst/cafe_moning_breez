# 날씨 자동수집 설정 (기상청 API)

날씨가 매출에 미치는 영향을 분석하려면 기상청 API 키가 필요합니다.
**키가 없어도 주문 시스템은 정상 동작**하며, 날씨 분석만 비활성화됩니다.

## 1. 기상청 API 키 발급 (무료, 약 5분)

1. **공공데이터포털** 접속: https://www.data.go.kr
2. 회원가입 / 로그인
3. 검색창에 **"기상청 단기예보 조회서비스"** 검색
4. 해당 서비스 → **"활용신청"** 클릭 (자동승인, 즉시 발급)
5. 마이페이지 → 데이터활용 → Open API → **인증키(서비스키)** 복사
   - 무료, 하루 100,000회 호출 가능 (카페엔 차고 넘침)

## 2. 카페 위치 좌표 찾기

1. 구글 지도에서 카페 위치 우클릭 → 위경도 숫자 복사
   (예: `37.5665, 126.9780`)
2. 앞이 위도(lat), 뒤가 경도(lon)

## 3. 설정 입력

`.cafe_config.json` 파일을 열어 아래 3개를 추가:

```json
{
  "admin_token": "기존값유지",
  "cafe_name": "우리 카페",
  "kma_service_key": "발급받은_인증키_붙여넣기",
  "cafe_lat": 37.5665,
  "cafe_lon": 126.9780
}
```

또는 환경변수로:
```bash
export KMA_SERVICE_KEY="발급키"
export CAFE_LAT="37.5665"
export CAFE_LON="126.9780"
```

## 4. 확인

서버 실행 후 검은 창에 30분 내 이런 줄이 뜨면 성공:
```
[날씨] 수집: Clear 22.5도
```

## 수집되는 데이터
- **하늘상태**: 맑음/구름/비/눈
- **기온** (섭씨)
- **습도** (%)

이 데이터가 주문에 자동으로 붙어, 대시보드의 "날씨별 매출" 분석에 쓰입니다.

## 작동 방식
- 30분마다 현재 날씨를 수집해 캐시
- 주문이 들어오면 그 시점 캐시된 날씨를 주문에 기록
- **과거 날씨는 소급 불가**하므로, 키를 빨리 넣을수록 데이터가 빨리 쌓임

## 주의
- 키 발급은 즉시 되지만, 가끔 실제 호출 가능까지 1시간~1일 걸릴 수 있음
- 키를 깃헙에 올리지 마세요 (`.cafe_config.json`은 .gitignore에 포함됨)
