# 깃헙에 푸시하는 법

cafe-os 코드를 https://github.com/waterfirst/cafe_moning_breez 에 올립니다.

## ⚠️ 먼저: 노출된 API 키 재발급
이전에 스크린샷으로 공유한 발전소 API 키가 노출됐습니다.
공공데이터포털 > 마이페이지에서 **재발급**하세요.
(이 코드는 키를 `.cafe_config.json`에만 넣고, 그 파일은 `.gitignore`로 깃에서 제외됩니다.)

---

## 방법 A: 스크립트 실행 (간단)

압축 푼 폴더에서 터미널 열고:
```bash
bash push_to_github.sh
```
- git 로그인이 안 돼 있으면 사용자명/비밀번호(또는 토큰)를 물어봅니다.
- 깃헙은 비밀번호 대신 **Personal Access Token(PAT)** 이 필요합니다.
  - 발급: 깃헙 > Settings > Developer settings > Personal access tokens
  - 권한: `repo` 체크
  - 토큰을 비밀번호 자리에 붙여넣기

## 방법 B: 직접 명령어 (수동)

```bash
cd cafe-os                     # 압축 푼 폴더로 이동
git init
git add -A
git commit -m "feat: cafe-os 초기 구현"
git branch -M main
git remote add origin https://github.com/waterfirst/cafe_moning_breez.git

# 기존 저장소에 이미지/README가 있으니 먼저 병합
git fetch origin main
git merge origin/main --allow-unrelated-histories -m "merge"

git push -u origin main
```

## 기존 저장소 내용은?
현재 저장소엔 이미지(11.png 등)와 README가 있습니다.
스크립트가 **자동 병합**하므로 기존 파일은 보존됩니다.
충돌 나면 안내 메시지대로 처리하세요.

## 푸시 후 확인
https://github.com/waterfirst/cafe_moning_breez 새로고침 →
`app/`, `db/`, `analytics/`, `docs/` 폴더가 보이면 성공.

## 참고: 왜 Claude가 직접 못 올리나?
보안 때문입니다. Claude가 당신의 깃헙 토큰을 받아 푸시하면
그 토큰이 노출될 위험이 있습니다. 그래서 코드만 준비하고,
실제 푸시(인증)는 당신 컴퓨터에서 하도록 했습니다.
