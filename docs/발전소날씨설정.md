# 발전소 기상 API 설정 (중부발전 — 선택)

> ⚠️ **일반 카페는 기상청 단기예보(날씨설정.md)를 쓰세요.**
> 이 발전소 API는 **카페가 발전소 인근(보령·서천·세종·제주·인천)**일 때만
> 의미가 있습니다. 일(日) 단위라 시간대별 분석은 안 됩니다.

## 기상청 vs 발전소 — 어느 걸 쓸까?

| 항목 | 기상청 단기예보 | 발전소 기상 |
|------|:-:|:-:|
| 위치 정밀도 | 읍·면·동 (카페 위경도) | 발전소 기준 |
| 시간 단위 | **시간대별** ✅ | 일 단위만 |
| 하늘상태(맑음/구름/비/눈) | ✅ | 강수만 추정 |
| 추가 항목 | 기온·습도 | 기온·습도·기압·풍속·일조시간 |
| 일반 카페 적합 | **✅ 권장** | 발전소 인근만 |

→ **카페가 발전소에서 멀면 기상청이 더 정확합니다.**

## 0. 인증키 재발급 (먼저!)
스크린샷에 키가 노출됐으면 **공공데이터포털 > 마이페이지 > 이 API > 인증키 재발급** 먼저 하세요.

## 1. stationName(발전본부 코드) 찾기
참고문서가 '복구중'이라 코드값이 공개돼 있지 않습니다. 직접 찾으세요:
1. 공공데이터포털 > 마이페이지 > Open API > 활용신청 현황 > 이 API
2. **"미리보기"** 또는 상세기능의 조회 테스트
3. stationName에 후보값을 넣어보고 응답이 오는 값 확인
   - 발전본부: 보령, 신보령, 신서천, 세종, 제주, 인천
   - 코드는 영문 약어(예: BR 등)일 가능성 — 미리보기로 확정

## 2. 설정 입력
`.cafe_config.json`:
```json
{
  "admin_token": "기존값",
  "weather_source": "komipo",
  "komipo_service_key": "재발급받은_Decoding_인증키",
  "komipo_station": "미리보기로_확인한_발전본부코드"
}
```
- `weather_source`를 `"komipo"`로 해야 발전소 소스를 씁니다.
- 기상청으로 되돌리려면 `"kma"` 또는 이 줄 삭제.
- 인증키는 **Decoding 키** 권장 (Encoding 키 쓰면 코드 수정 필요).

## 3. 확인
서버 실행 후 검은 창에:
```
[날씨/komipo] 수집: Rain 24.5도
```

## 한계 (알고 쓰세요)
- **일 단위**: "16시 피크 때 날씨"는 못 봅니다. 일별 매출-날씨 분석만 가능.
- **하늘상태 없음**: 맑음/구름 구분 불가. 강수량으로 비(Rain)만 추정.
- **전날 데이터**: 당일 집계 전이라 어제 기준으로 가져옵니다.

## 추천
가능하면 **기상청과 병행**하거나 기상청을 메인으로 쓰세요.
발전소 API의 장점인 일조시간·기압·풍속이 카페 분석에 꼭 필요한 게 아니라면,
시간대별이 되는 기상청이 분석 가치가 더 높습니다.
